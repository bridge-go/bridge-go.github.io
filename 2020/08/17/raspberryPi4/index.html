由于主力机由树莓派3迁移到树莓派4B，因此重开一个文档记录

安装Raspbian系统

根据官网说明，安装Raspberry Pi Imager, 选择安装的系统（一般安装推荐的系统即可，也可以安装Ubuntu等系统），选择要写入的SD卡，会自动写入系统。

树莓派的系统与硬件是分离的，同一张卡可以拔下来给不同的硬件使用，没有太大区别。

网络连接

网络来源

WiFi

树莓派会记忆网络连接，下次会自动连接。对于校园网的WiFi，也会自动实现网页登录，无需单独打开网页。

在有显示器的情况下十分方便。

Ethenet

可以通过网线直接连接，无法登录校园网，无网络连接，但在学校可以获得校内网IP。无需显示器等外设。

电脑与树莓派通过网线连接

通过一根网线将笔记本的网络共享给树莓派：

开始 -> 控制面板 -> 网络和Internet -> 网络和共享中心 -> 查看网络状态和任务 -> 更改适配器设置

找到笔记本已经连接的WiFi，比如ZJUWLAN，右键属性 -> 共享选项卡 -> 允许其他网络用户通过此计算机的Internet连接来连接（同时需要在家庭网络连接里选上自己的树莓派端口）

该端口可以通过插拔网线识别，后续可以更名为RPi进行标记

电脑热点分享网络给树莓派

步骤同上，区别是选择树莓派连接端口时不选择以太网，选择自己热点的名称。

如未重命名过，可能是本地连接 *n，此处我的笔记本改为了SHARED WLAN，共享网络时选择这个即可

SSH连接

SSH连接唯一重要的东西就是树莓派的IP地址

ssh usrname@ip_address -p portnum

树莓派用默认端口22就可以，因此可以简化为 ssh pi@ip_address

WiFi

WiFi直连时，可以通过屏幕等外设先获取IP，再进行SSH连接，但IP会不定期变化，导致连接断开。网上翻了非常多的资料，最后的结论就是校园网内无法设置为静态IP，如果静态IP信息和ZJUWLAN分配的不一致，就无法成功登陆网络，那就无法和笔记本在同一局域网内，无法进行SSH连接。

Ethernet

此时无法直接获取树莓派IP，如果需要SSH连接，需要先通过屏幕获取IP地址。

获取IP后需要在内网条件下才能成功进行SSH连接，不在局域网内无法连接。

电脑与树莓派通过网线连接

可以共享网络，可以根据主机网络分配树莓派IP，是否可以设置静态IP未尝试

电脑热点分享网络给树莓派

可以共享网络，主机给树莓派分配IP。
